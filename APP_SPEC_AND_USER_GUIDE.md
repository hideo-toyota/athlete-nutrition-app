# Athlete Nutrition App 仕様書・使い方・活用方法

## 1. アプリ概要

Athlete Nutrition App は、試合・大会・バレエ発表会などの本番日から逆算して、競技・体重・疲労・睡眠・食欲に合わせた食事計画、実行記録、振り返り、周囲への共有までを1画面で支援するコンディショニングアプリです。

一般的な食事管理アプリのようにカロリーや栄養素を記録するだけでなく、以下の問いに答えることを目的にしています。

- 本番まであと何日で、今日から何を増やすべきか
- 今日の競技・練習内容に対して、糖質やたんぱく質は足りているか
- 試合や本番日から逆算して、何日前から何を増やすべきか
- 疲労感、睡眠、食欲、体重の変化から、補食や回復をどう調整すべきか
- 保護者、コーチ、チームメイトは、今日何を支援すればよいか

## 2. 想定ユーザー

- 部活動やクラブチームで競技に取り組む学生アスリート
- 食事や補食を支える保護者
- 選手の状態を把握したいコーチ
- 複数競技の選手を見るトレーナー、栄養士
- Vibe Coding / Engineering 枠で提出するプロトタイプの審査者

## 3. 解決したい課題

アスリートの食事管理では、理想的な栄養計算だけでは継続しにくい課題があります。

- 試合前日、試合当日、回復日で見るべき栄養指標が変わる
- 疲れている日ほど食欲が落ち、理想の食事を実行できない
- 練習前後の補食が抜けても、後から原因を振り返りにくい
- 保護者やコーチが、細かい栄養計算ではなく「今日の支援ポイント」を知りたい
- 仲間同士で継続を支え合う仕組みが少ない

このアプリでは、栄養計算、実行チェック、体調ログ、CtoC支援、共有メモを組み合わせ、現場で使いやすい形に寄せています。

## 4. 主要機能

### 4.1 Sport Profile

競技を選ぶと、競技特性に応じた栄養目標と注意点が表示されます。

対応競技:

- 陸上・ランニング
- 野球
- サッカー
- バレエ
- バスケットボール
- ゴルフ
- 水泳
- 卓球

表示内容:

- 糖質目標
- たんぱく質目標
- 重点栄養素
- 水分補給メモ
- 競技別リスクメモ
- 参考エビデンス

### 4.2 Day Mode

日ごとの目的に応じて、栄養目標の見方を切り替えます。

- 通常日
- 練習日
- 試合前日
- 試合当日
- 回復日

Day Mode はカレンダーの日付ごとに保存されます。

### 4.3 Calendar

月間カレンダーで、日別の記録状況を確認できます。

確認できるもの:

- 選択日
- 今日の日付
- Day Mode
- 食事プランの完了数
- 本番日マーク

### 4.4 Food Plan

朝食、昼食、補食、夕食などの食事プランを作成・編集できます。

記録項目:

- 食事名
- タイミング
- kcal
- 糖質
- たんぱく質
- 実行状態
- 写真

実行状態:

- 未実施
- 一部だけ
- 完了
- スキップ

### 4.5 Meal Photo

食事カードに写真を添付できます。

用途:

- 実際に食べたものを文字だけでなく視覚的に残す
- 保護者やコーチと食事内容を共有しやすくする
- あとから食事量や内容を振り返る

写真は圧縮された thumbnail data として localStorage 内の plan に保存されます。

### 4.6 Condition Log

日別にコンディションを記録できます。

記録項目:

- 睡眠時間
- 疲労感
- 食欲
- 体重
- メモ

これらの値は AI Coach Notes と Event Fueling の補正に使われます。

### 4.7 Event Fueling / 本番逆算

試合、大会、発表会などの本番日を登録すると、本番日から逆算して日別の栄養目標を表示します。

入力項目:

- 本番名
- 日付
- 種別
- 競技時間
- 優先度

表示内容:

- 初回ユーザー向けの Action Path
- 本番までの日数
- 糖質目標 g/day
- 糖質目標 g/kg/day
- たんぱく質目標 g/day
- たんぱく質目標 g/kg/day
- カーボローディング期の説明
- 本番当日の補給タイミング
- 翌日の回復栄養
- 疲労、睡眠、食欲による補正コメント
- 選択日の朝食、昼食、補食、夕食プラン自動作成

栄養計算の考え方:

- 体重あたりの糖質・たんぱく質目標を基本にする
- 本番3日前から前日は、競技時間に応じてカーボローディング寄りにする
- 本番当日は、消化しやすい糖質と競技中補給を重視する
- 翌日は、糖質、たんぱく質、水分補給による回復を重視する
- 疲労が高い日は糖質下限を少し上げる
- 食欲が低い日は、ゼリー、おにぎり、ヨーグルトなど実行しやすい形を優先する
- 自動作成する食事プランは、1日の糖質・たんぱく質目標を朝食、昼食、補食、夕食に配分する
- 選手本人向けには「今日やること」、保護者・コーチ向けには「支援すること」を短く分けて表示する

### 4.8 AI Coach Notes

食事プラン、実行状態、体調ログ、過去データからコメントを表示します。

検出する例:

- 糖質ターゲット不足
- たんぱく質不足
- 補食スキップ
- 睡眠不足
- 疲労感の高さ
- 食欲低下
- 過去7日の実行率
- 3日連続の糖質不足
- 2週間の体重変化
- 本番日までの逆算コメント

現時点では LLM API ではなく、ルールベースの MVP です。

### 4.9 CtoC Support

本人だけでなく、仲間と支え合うための機能を入れています。

Buddy Check:

- 自分とバディの実行率、睡眠、疲労を比較
- 未実施の食事数を確認

Pair Score:

- 直近7日平均のペア達成率
- 連続達成日数
- 今日の支援ポイント

Cheer Stamps:

- ナイス補食
- 水分OK
- 回復しよう
- 明日いける

Team Food Log:

- チームメイトの食事ログを確認
- 似た食事を自分のプランに取り込む
- リマインドに追加する

Reminder Scheduler:

- 自分やバディに補食、水分、回復の声かけを予約

### 4.10 Guardian Share Mode

保護者やコーチ向けの共有メモを生成します。

共有される内容:

- 日付
- 競技
- Day Mode
- 睡眠、疲労、食欲、体重
- 栄養目標
- 食事進捗
- 支援してほしいこと
- 代替候補

## 5. 基本的な使い方

### 5.1 初回確認

1. アプリを開きます。
2. 画面上部の BootCamp 提出用パネルで、アプリ概要を確認します。
3. 右側の Athletes から選手を選びます。
4. Sport Profile で競技が合っているか確認します。
5. Condition Log で睡眠、疲労、食欲、体重を入力します。

### 5.2 日々の食事記録

1. Calendar で記録したい日付を選びます。
2. Day Mode を選びます。
3. 既存の食事プランを確認します。
4. 必要に応じて Add Plan から食事を追加します。
5. 食べたら `完了`、一部だけなら `一部だけ`、食べられなければ `スキップ` にします。
6. 写真を追加すると、実際の食事内容を残せます。
7. AI Coach Notes で不足や改善ポイントを確認します。

### 5.3 本番前の使い方

1. Event Fueling / 本番逆算 に本番日を登録します。
2. 本番までの日数を確認します。
3. 日別の糖質・たんぱく質目標を確認します。
4. 必要な日付を開き、`選択日の食事プランを作る` を押します。
5. 作成された朝食、昼食、補食、夕食カードを実行しやすい内容に調整します。
6. 当日は、消化しやすい糖質と補給タイミングを確認します。
7. 翌日は、回復栄養と水分補給を優先します。

### 5.4 仲間と使う場合

1. Buddy Check で自分とバディの状態を確認します。
2. Pair Score で直近7日の流れを見ます。
3. Cheer Stamps で軽い応援を送ります。
4. Team Food Log で仲間の食事例を見ます。
5. 良さそうな食事は `真似する` で自分のプランに取り込みます。
6. 必要なら `リマインドに追加` します。

### 5.5 保護者・コーチと共有する場合

1. 食事と体調を入力します。
2. AI Coach Notes を確認します。
3. Share Panel で共有メモを生成します。
4. コピーして、LINE、Slack、メールなどに貼り付けます。

## 6. 活用シーン

### 6.1 部活動の試合週

本番日を登録し、3日前から主食量を少しずつ増やす計画を確認します。選手本人は食事チェック、保護者は補食準備、コーチは疲労感や睡眠不足の兆候を確認できます。

### 6.2 遠征や連戦

Team Food Log と Convenience Suggestions を使い、コンビニで実行しやすい代替案を確認します。補食スキップが続く場合は AI Coach Notes が警告します。

### 6.3 食欲が落ちた日

Condition Log で食欲を低めに入力すると、AI Coach Notes や Event Fueling で、飲めるもの、柔らかい糖質、小分け補給を優先する提案が出ます。

### 6.4 保護者のサポート

保護者は細かい栄養計算ではなく、Share Panel の「今日支援してほしいこと」を見て、補食、水分、夕食の方向性を支援できます。

### 6.5 チーム内の継続支援

Buddy Check や Cheer Stamps により、食事記録を一人で抱えず、仲間同士で小さく支え合えます。

## 7. デモ手順

応募や発表で見せる場合は、以下の順番が伝わりやすいです。

1. 画面上部の BootCamp 提出用パネルで「本番逆算型」のコンセプトを見せる
2. Event Fueling に5日後の試合・大会・バレエ発表会を登録する
3. Sport Profile で競技を切り替え、競技別の目標が変わることを見せる
4. Condition Log で疲労感を高め、睡眠や食欲も入力する
5. Action Path とカーボローディング、g/kg の日別目標を見せる
6. `選択日の食事プランを作る` で食事カードへ変換する
7. 疲労・睡眠・食欲による補正コメントを見せる
8. 食事カードの実行状態を変更し、計画倒れを記録できることを見せる
9. AI Coach Notes のコメントを見せる
10. Share Panel で保護者・コーチ向け共有メモを見せる
11. Buddy Check / Team Food Log / Reminder は「継続支援の補助機能」として短く見せる

## 8. データ仕様

このプロトタイプは localStorage に保存します。

主な保存キー:

- `plans`: 食事プラン
- `athletes`: 選手プロフィール
- `athlete-checkins`: 日別コンディションログ
- `athlete-day-modes`: 日別 Day Mode
- `athlete-events`: 本番日情報
- `selected-athlete-id`: 最後に選択した選手
- `ctoc-reminders`: リマインド予約

写真は `plans` の中に圧縮した data URL として保存されます。

## 10. 栄養ロジックの考え方

栄養目標は、競技、Day Mode、体重、本番までの日数、疲労、睡眠、食欲をもとに簡易計算します。

ベース:

- 糖質: 競技と日別目的に応じた g/kg/day
- たんぱく質: 競技別の g/kg/day
- 本番前: カーボローディング寄りに補正
- 本番当日: 消化しやすい糖質と競技中補給を意識
- 翌日: 回復栄養と水分補給を重視

参考:

- Academy of Nutrition and Dietetics / ACSM / Dietitians of Canada: Nutrition and Athletic Performance
- International Society of Sports Nutrition: Protein and Exercise
- 各競技別レビュー文献

注意:

このアプリの値は教育・デモ用途の目安です。実際の栄養指導では、年齢、性別、体組成、既往歴、血液検査、発汗量、競技レベルなどを含めた専門的評価が必要です。

## 12. 応募時の使い方

応募フォームには、以下のどれか1つ以上のURLを入れます。

- 公開GitHubリポジトリ
- 公開WebデモURL
- デモ動画URL

フォームの「あなたの実績や活動内容がわかるもの」には、`BOOTCAMP_APPLICATION.md` の文章をベースに貼り付けます。

公開前に確認すること:

- リポジトリが public になっている
- README に概要、使い方、検証結果がある
- この仕様書が含まれている
- 不要な検証データや個人情報が含まれていない
- 公開URLを第三者が開ける
